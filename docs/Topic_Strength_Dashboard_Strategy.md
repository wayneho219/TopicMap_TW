## 【1】儀表板的核心目的

### 1.1 解決的投資痛點

| 痛點 | 傳統方案的限制 | 本儀表板的解決方案 |
|------|----------------|--------------------|
| **訊息超載** | 每日數百則新聞，難以識別真正重要的題材 | 自動聚類+強度排序，一眼看出市場焦點 |
| **滯後性** | 事後分析，無法及時捕捉新興題材 | 實時新聞流+每日更新，搶先發現 |
| **主觀判斷** | 依賴分析師經驗，缺乏量化依據 | 數據驅動的強度指數，可溯源驗證 |
| **題材轉移視盲** | 無法判斷題材何時由熱轉冷 | MoM/WoW 趨勢線，清晰展示強度曲線 |
| **個股影響評估困難** | 不知道一個題材涉及多少股票 | 覆蓋度(Coverage)指標，量化關聯性 |
| **情緒與事實混淆** | 無法區分「炒作」與「實質利多」 | 情緒分數+新聞頻率分離，雙維度評估 |

### 1.2 使用者與應用場景

#### **A. 資產管理公司投資經理**
- **目的**：尋找「正在升溫」的產業主題作為選股信號
- **決策**：題材排名 Top 5 + 關聯個股 → 超額收益機會
- **時間頻率**：每日開盤前檢視，周報納入行業分析

#### **B. 研究員 / 分析師**
- **目的**：追蹤供應鏈、科技趨勢等主題的演進
- **決策**：題材時序圖 + 關鍵詞詞雲 → 深度研究主軸
- **時間頻率**：定期監控，撰寫研究報告時核對事實依據

#### **C. 交易員 / 日內操盤手**
- **目的**：捕捉題材異常波動作為短期交易訊號
- **決策**：異常警示(Spike Detection) + 情緒指數 → 博弈題材
- **時間頻率**：盤中實時監控，5-15 分鐘刷新一次

#### **D. 基金經理 / 投資顧問**
- **目的**：向客戶解釋投資組合中的「題材曝露」
- **決策**：投資組合×題材矩陣，展示組合對各主題的敏感度
- **時間頻率**：月度報告、季度檢視

## 【2】主題強度（Topic Strength）的定義與計算方式

### 2.1 主題強度的核心邏輯

**主題強度** = 一個投資題材在特定時間段內對市場的「關注度 + 熱度 + 影響力」的綜合量化指標。

> **簡單類比**：就像「病毒傳播指數」結合「病例數+變異速度+感染範圍」，主題強度結合「新聞量+趨勢動量+涉及個股數」。

---

### 2.2 計算公式（3 個版本，漸進式複雜度）

#### **版本 A：基礎版（Baseline）**
適用於 MVP、快速上線

```
TopicStrength_A(t) = 
    α × Frequency(t) 
    + β × TrendMomentum(t) 
    + γ × Coverage(t)

其中：
  α = 0.4, β = 0.35, γ = 0.25   （權重係數，可根據回測調整）
  
  Frequency(t) = 過去 7 天新聞篇數 / 該主題歷史平均篇數
                （正常化至 [0, 1] 區間）
  
  TrendMomentum(t) = (今日篇數 - 7日平均篇數) / 7日標準差
                     （Z-score，捕捉「加速度」）
  
  Coverage(t) = min(涉及個股數 / 上市公司總數, 1.0)
                （覆蓋率，標準化至 [0, 1]）
```

**優點**：易於實作、計算快速、可解釋性強  
**缺點**：未考慮情緒、時間衰減、關鍵詞權重

---

#### **版本 B：進階版（Enhanced）**
適用於生產環境

```
TopicStrength_B(t) = 
    α × Frequency_Normalized(t) 
    + β × TrendMomentum(t) 
    + γ × Coverage_Weighted(t)
    + δ × SentimentScore(t)
    + ε × KeywordImportance(t)

其中：

① Frequency_Normalized(t)
   = ∑[ count(article_i) × TimeDecay(t_i) ] / Historical_Avg
     (所有貼文數 × 時間衰減因子，除以歷史平均)
   
   TimeDecay(t_i) = exp(-(t - t_i) / τ)
                    τ = 7 天（時間常數）
   
   → 最近的新聞權重高，14 天前的新聞權重僅剩 6.25%

② TrendMomentum(t)
   = [MA(1d) - MA(7d)] / MA(7d) × 100%
     (日均線 vs 周均線，捕捉「加速」訊號)

③ Coverage_Weighted(t)
   = ∑[ (stock_price_change_pct × article_count) ] / ∑article_count
     (加權平均：高漲幅的股票權重更高)
   
   → 如果涉及 10 檔股票，但只有 3 檔上漲，實際覆蓋度打折

④ SentimentScore(t)
   = (正向新聞數 - 負向新聞數) / 總新聞數
     (範圍：[-1, 1]，經過 softmax 正規化至 [0, 1])
   
   → 需搭配 BERT-base-chinese 或開源台灣金融情緒模型

⑤ KeywordImportance(t)
   = 主題內關鍵詞的平均 TF-IDF 分數
   = ∑(TF-IDF_i × Frequency_i) / ∑Frequency_i
   
   → 越多人提到「台積電 + 3 奈米」，就越重要

權重建議：α=0.3, β=0.25, γ=0.2, δ=0.15, ε=0.1

完整強度指數（0-100）：
TopicStrength_B(t) = round( TopicStrength_B(t) × 100 )
```

**優點**：綜合多維度、更接近真實投資價值  
**缺點**：需要情緒模型訓練、計算略複雜

---

#### **版本 C：專業版（Professional）**
適用於 B2B 高端客戶

```
TopicStrength_C(t) = 
    [ Market_Component(t) 
      + News_Component(t) 
      + Sentiment_Component(t) 
      + Anomaly_Component(t) ] 
    × Decay_Factor(t)

──────────────────────────────────────────

① Market_Component(t) 
   = ∑[ Correlation(topic_i, stock_j) × Return_j(t) ] / N_stocks
   
   → 實時計算主題內個股的「平均報酬率」
   → 主題強度與實際個股表現掛鉤（因果驗證）

② News_Component(t)
   = Frequency(t) × TrendAcceleration(t) × KeywordNovelty(t)
   
   TrendAcceleration(t) = (Δ篇數 / Δt) / Historical_Volatility
                          (新聞增速 / 歷史波動率，標準化)
   
   KeywordNovelty(t) = ∑[ IDF_i × IsNewKeyword_i ] / Total_Keywords
                       (新出現的關鍵詞佔比——捕捉「新題材」)

③ Sentiment_Component(t)
   = AggregatedSentiment(t) × SentimentConfidence(t)
   
   SentimentConfidence = Positive_Articles / Total_Articles
                         (確信度：多數人同意該題材好壞)

④ Anomaly_Component(t)
   = max(0, (Actual_Frequency(t) - Predicted_Frequency(t)) / σ_prediction)
   
   → 基於 ARIMA/Prophet 預測，偵測異常波動
   → 如果今天新聞數超預測 3σ，則異常指數 = 3

⑤ Decay_Factor(t)
   = ∏[ (1 - decay_rate)^(days_ago) ] for all articles
   
   → 30 天前的主題對現在的貢獻度大幅下降
   → 避免「殭屍題材」持續得分

完整公式：
TopicStrength_C(t) = normalize( 
                        0.35×Market 
                        + 0.35×News 
                        + 0.20×Sentiment 
                        + 0.10×Anomaly 
                      ) × TimeDecay
                    ∈ [0, 100]

──────────────────────────────────────────

實作建議：
• Market_Component 需要 5 分鐘更新一次（需實時股價 API）
• News_Component 每小時更新一次（新聞爬蟲頻率）
• Sentiment 與 Anomaly 每日更新一次（訓練開銷）
```

**優點**：最全面、可用於 ML 模型訓練（預測個股收益）  
**缺點**：實現複雜、需多個外部 API、計算成本高

### 3.1 各 KPI 的詳細說明與用途

#### **KPI 1️⃣：主題強度指數（Topic Strength Index）**
```
定義：0-100 的加權綜合指標，代表該題材在當前時刻的「市場熱度」

用途：
  • 排序題材，決定儀表板首頁的 Top 5 排行
  • 觸發警示：若指數環比上升 50%，發送推播提醒
  • 投資組合風險評估：我的持股曝露在哪些強勢題材上？

解讀舉例：
  • 指數 80+ ：極熱，短期可能衝高但風險大（高波動）
  • 指數 50-79：溫熱，新興題材，值得深入研究
  • 指數 20-49：溫和，題材趨勢不明
  • 指數 <20  ：冷卻，可能進入衰退期，謹慎進場
```

---

#### **KPI 2️⃣ & 3️⃣ & 4️⃣：新聞頻率 + WoW + MoM**
```
定義：量化新聞提及的「絕對值」和「相對變化」

用途：
  • WoW（周環比）：快速捕捉「新興熱點」
    例：某題材 WoW = +200%，說明最近 1 週快速升溫
  
  • MoM（月環比）：判斷「長期趨勢」是否健康
    例：某題材 MoM = -30%，說明月度熱度在下滑
  
  • 組合判斷：WoW 高 + MoM 低 = 短期熱點，可能無持久力
           WoW 高 + MoM 高 = 長期熱點，多頭信號強

指標設計：
  篇數 = ∑ count(articles) 過去 N 天
  WoW = (Week1 - Week0) / Week0 × 100%  ← 捕捉「加速度」
  MoM = (Month1 - Month0) / Month0 × 100%  ← 捕捉「趨勢」
```

---

#### **KPI 5️⃣：關鍵詞權重（Top 5 Keywords + TF-IDF）**
```
定義：該主題內最重要、最頻繁的 5 個關鍵詞及其相對重要性

計算：
  TF-IDF(keyword_i) = TF(i, topic) × IDF(i, all_topics)
  
  TF = 詞在主題內的頻率
  IDF = log(總主題數 / 包含該詞的主題數)  ← 稀有詞權重高

用途：
  • 視覺化「詞雲」：面積大 = 權重高
  • 快速理解題材：一眼看 Top 5 詞就知道在講什麼
  • 題材漂移偵測：如果關鍵詞組突然大幅變化，可能是題材演變
  
舉例：
  主題「能源轉型」的 Top 5 詞：
    1. 太陽能 (TF-IDF: 0.85)
    2. 風電 (0.78)
    3. 儲能電池 (0.72)
    4. ESG (0.68)
    5. 碳中和 (0.65)
```

---

#### **KPI 6️⃣：情緒指數（Sentiment Index）**
```
定義：-100 ~ +100 之間，表示該主題新聞的整體正負面情緒

計算方式：
  情緒指數 = (正面新聞數 - 負面新聞數) / 總新聞數 × 100
  
  需引入 BERT-base-chinese / 開源台灣金融情緒模型
  或人工標記訓練集後微調

用途：
  • 風險預警：高熱度 + 負面情緒 = 可能陷阱題材
  • 投資時機：上漲題材 + 低情緒 = 被低估的機會
  • 情緒過度：情緒指數 +90 = 市場極度樂觀，可能反轉
  
舉例：
  • 「高端疫苗」題材：高熱度 + 情緒 -80 = 避開
  • 「輝達併購」題材：高熱度 + 情緒 +85 = 可能有補漲空間
```

---

#### **KPI 7️⃣：覆蓋個股數（Stock Coverage）**
```
定義：該主題關聯的不同上市公司家數

計算：
  COUNT(DISTINCT stock_id WHERE topic_id = X AND association_score > threshold)

用途：
  • 分散度評估：覆蓋 50+ 股 = 分散，風險低但上升動能小
                覆蓋 5-10 股 = 集中，風險高但收益大
  
  • 投資組合匹配：若我持 3 檔股票都在同一主題，曝露過度
  
  • 題材強度修正：若只涉及 1-2 家龍頭股，實際覆蓋度要折扣
  
舉例：
  • 「台積電熱潮」：覆蓋 1 股 = 單一風險，強度應折 50%
  • 「AI 晶片」：覆蓋 15-20 股 = 行業級別主題，強度有說服力
```

---

#### **KPI 8️⃣：平均個股漲幅 (%)（Average Return）**
```
定義：覆蓋範圍內所有個股的平均報酬率

計算：
  AvgReturn(topic, period) = AVG( Return_i ) 
                              for all stocks in topic
  
  Return_i = (Price_today - Price_t) / Price_t × 100%

用途：
  • 「題材有效性檢驗」：若強度指數 80 但個股都在跌，模型需調整
  • 超額收益預期：歷史上，強度 70+ 的題材次日平均漲幅 +2.5%
  • 風險調整收益：結合波動率，計算 Sharpe Ratio
  
時間維度：
  • 1D 漲幅：盤中實時，最敏感
  • 5D 漲幅：週線級別，過濾雜訊
  • 20D 漲幅：月線級別，判斷趨勢健全度
  
舉例：
  「電動車供應鏈」主題：
    • 1D 平均漲幅 = +1.8%
    • 5D 平均漲幅 = +5.2%
    • 20D 平均漲幅 = +12.5% ← 趨勢確立
```

---

#### **KPI 9️⃣：異常分數（Spike Detection Score）**
```
定義：當前新聞量相對「統計預測值」的偏離程度，單位：標準差 (σ)

計算方式：
  Spike_Score(t) = (Actual_Count(t) - Predicted_Count(t)) / σ_prediction
  
  其中 Predicted_Count 基於：
    • ARIMA 模型：用過去 60-90 天新聞量預測未來
    • 或 Prophet：自動偵測季節性、節假日效應
  
  解釋：
    Spike_Score > +2σ  = 異常高漲（95% 信心）
    Spike_Score > +3σ  = 極度異常（99.7% 信心，黑天鵝級別）
    Spike_Score < -2σ  = 異常低迷

用途：
  • 「新聞炸裂」警示：當 Spike_Score > +3 時，立即推播
  • 黑天鵝檢測：突發事件通常表現為 Spike_Score 激增
  • 題材反轉訊號：若 Spike_Score 長期為負，題材可能已死
  
實務例：
  • 2024/3/15 台積電法說會：Spike_Score 突升至 +4.5σ
  • 正常 AI 題材：Spike_Score 在 -1σ ~ +1σ 波動
```

---

#### **KPI 🔟：題材生命週期階段（Lifecycle Stage）**
```
定義：根據新聞量曲線的數學特徵，判斷題材處於哪個階段

四個階段劃分：
  
  ① 萌芽期（Emerging）
     特徵：新聞量快速上升（二階導數 > 0），但絕對量還小
     訊號：WoW > +80%, 新聞數 < 歷史中位數 50%
     投資策略：潛在黑馬，但高風險
  
  ② 成長期（Growing）
     特徵：新聞量快速上升且絕對量大（二階導數 > 0, 一階導數 > 0）
     訊號：WoW > +30%, 新聞量 = 歷史上分位 60-80%
     投資策略：建議進場，買需求旺盛
  
  ③ 成熟期（Mature）
     特徵：新聞量平穩（二階導數 ≈ 0，一階導數 ≈ 0）
     訊號：WoW 接近 0%, 新聞量高位持穩
     投資策略：謹慎持有，警惕反轉訊號
  
  ④ 衰退期（Declining）
     特徵：新聞量開始下降（一階導數 < 0）
     訊號：WoW < -20%, Spike_Score 持續負值
     投資策略：逐步減倉，準備轉向

階段判定演算法：
  Stage = classify(
    WoW, MoM,           ← 趨勢
    News_Volume_Percentile,  ← 絕對量
    Second_Derivative   ← 加速度
  )
  
舉例：
  主題「AI 晶片」在過去 30 天：
    第 1-10 天：Stage = Emerging (WoW +150%)
    第 11-20 天：Stage = Growing (WoW +45%)
    第 21-30 天：Stage = Mature (WoW +5%)
    → 預測第 31-45 天可能轉為 Declining
```

#### **KPI 1️⃣1️⃣：投資人提及度（Social Mentions）[可選]**
```
定義：社群媒體（PTT、Dcard、股市爆料公社等）對該主題的討論熱度

計算：
  Social_Mentions = ∑[ posts containing topic_keywords ] 
                    在 24 小時內
  
  Sentiment_from_Social = (推文數 - 噓文數) / 總迴應數

用途：
  • 散戶情緒對沖：機構看漲但散戶看跌 = 尋求共識
  • FOMO 偵測：社群討論激增 +200% = 可能追高區
  • 題材接納度：若機構新聞量高但社群提及度低 = 題材曲高和寡
  
注意事項：
  • 社群數據「雜訊」多，需清洗機器人帳號
  • 需定期更新爬蟲規則（平台演變）
  • 隱私和爬蟲協議風險，需法律評估
  
此 KPI 可在 MVP 後作為加值功能後續添加。
```

---

#### **KPI 1️⃣2️⃣：預測下週排名（Rank Prediction）**
```
定義：基於當前強度趨勢和時序模型，預測該主題下週的排名

計算方式：
  Predicted_Strength(next_week) = Prophet.forecast(
    historical_strength_timeseries,
    periods=7
  )
  Predicted_Rank = rank( Predicted_Strength ) among all topics

用途：
  • 提前佈局：若預測下週排名躍升 3 位，可趁現在埋伏
  • 風險規避：若預測排名下滑，提前準備轉向
  • 研究員報告時程安排：下週會熱的主題，本週應寫深度報告
  
精準度：
  • 1 週預測：準度約 65-75%（時序模型標準水準）
  • 2-4 週預測：準度 50-65%（不確定性上升）
  
此 KPI 需至少 3-6 個月的歷史數據訓練才可上線。
```

#### **模組 10：主題生命週期預測（Lifecycle Forecast）**

```

預測模型：
  • 輸入：主題歷史新聞量時序、關鍵詞演變、外部訊號
  • 輸出：未來 4 週的生命週期預測 + 信心度
  
視覺展示：

  階段     當前    1週    2週    3週    4週
  ────────────────────────────────────────
  Emerging  ✓                            
  Growing        ✓      (85%)
  Mature              ✓      (60%)
  Declining                    ✓    (45%)
  
  讀法：
    • 當前處於 Growing 期（確定）
    • 有 85% 概率在 2 週內進入 Mature 期
    • 有 60% 概率在 3 週內進入 Mature 期
    • 有 45% 概率在 4 週內開始衰退

建議：
  • 根據預測，「強度 Top 3」主題中 2 個預計 2 週後衰退
  • 建議提前調整持倉，避免套牢
```
